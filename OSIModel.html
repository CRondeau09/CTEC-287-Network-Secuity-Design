OSI Model - 7 Layers

Open Systems Interconnection Reference Model

Memory Device: "Please Do Not Throw Sausage Pizza Away"
Physical → Data Link → Network → Transport → Session → Presentation → Application
7

Application Layer

User Interface

The closest layer to the end user. Provides network services directly to applications and end-users. This is where user interaction with the network begins.

Common Protocols:

HTTP/HTTPS FTP SMTP POP3 IMAP DNS DHCP Telnet SSH

Real-World Examples:

Web browsers, email clients, file transfer applications, remote desktop software
6

Presentation Layer

Data Translation & Encryption

Translates data formats, handles encryption/decryption, and compression/decompression. Ensures data from the application layer can be read by the receiving system.

Functions & Protocols:

SSL/TLS JPEG GIF PNG MPEG ASCII EBCDIC

Real-World Examples:

Data encryption, image/video compression, character encoding conversion
5

Session Layer

Session Management

Establishes, manages, and terminates connections between applications. Controls dialogues and keeps sessions separate.

Common Protocols:

NetBIOS RPC PPTP L2TP SQL Sessions

Real-World Examples:

Login sessions, database connections, video conferencing coordination
4

Transport Layer

End-to-End Communication

Provides reliable data transfer services. Handles error detection, flow control, and data segmentation. Ensures complete data transfer.

Common Protocols:

TCP UDP SPX

Key Functions:

Segmentation, error correction, flow control, port addressing, reliable delivery
3

Network Layer

Routing & Path Determination

Routes data packets between different networks. Determines the best path for data to travel from source to destination.

Common Protocols:

IP ICMP OSPF BGP RIP IPX

Key Functions:

IP addressing, routing, packet forwarding, path determination
2

Data Link Layer

Node-to-Node Communication

Provides error-free transfer of data frames between two directly connected nodes. Handles MAC addressing and error detection.

Common Protocols:

Ethernet Wi-Fi (802.11) PPP Frame Relay ATM

Key Functions:

MAC addressing, frame synchronization, error detection, flow control
1

Physical Layer

Hardware & Transmission

Transmits raw bit streams over physical media. Defines electrical, mechanical, and timing specifications for network hardware.

Physical Media & Standards:

Ethernet Cables Fiber Optic Wireless Radio Bluetooth USB

Real-World Examples:

Cables, hubs, repeaters, network adapters, wireless access points

Data Flow Direction

Sending: Application (7) → Physical (1)
↓
Receiving: Physical (1) → Application (7)
↑

Each layer adds/removes its own header information (encapsulation/decapsulation)
